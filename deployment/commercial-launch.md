# 新源3D赛车国内商业化上线手册

## 当前定位

- GitHub Pages 继续作为测试、展示和版本预览环境。
- 正式商业入口面向中国大陆用户，使用国内云厂商静态托管和 CDN。
- 第一版商业化采用免费试玩加广告预留，不接账号、支付或云排行榜。

## 推荐生产架构

首选：阿里云 OSS 静态网站托管 + CDN。

备选：腾讯云 COS 或静态网站托管 + CDN。

生产资源保持静态化：

- `index.html`
- `sw.js`
- `assets/audio/start.mp3`
- `assets/audio/race.mp3`

仓库仍作为源码中心。生产发布包由 `scripts/build-production.ps1` 生成到 `dist/`，再上传到 OSS/COS。

## 域名与备案

1. 购买正式域名。
2. 使用大陆节点前完成 ICP 备案。
3. 备案完成后把域名解析到 CDN。
4. CDN 开启 HTTPS。
5. CDN 回源到 OSS/COS 静态网站源站。

备案完成前可以继续使用 GitHub Pages 或香港节点测试，但不要当作商业主站。

## 发布流程

1. 在本地确认游戏可运行。
2. 生成生产包：

   ```powershell
   .\scripts\build-production.ps1
   ```

3. 上传 `dist/` 目录里的全部内容到 OSS/COS。
4. 刷新 CDN 缓存：
   - `/index.html`
   - `/sw.js`
   - `/assets/audio/start.mp3`
   - `/assets/audio/race.mp3`
5. 用版本参数验证：

   ```text
   https://你的域名/?v=提交号
   ```

## 广告预留策略

当前页面已经预留两个隐藏广告位：

- `#ad-start`：开始界面广告位。
- `#ad-gameover`：结算界面广告位。

第一阶段不要接入广告 SDK，避免影响首屏加载和审核。等访问稳定、留存数据明确后，再选择广告平台并接入。

## 手机访问验收

每次生产发布后至少测试：

- 中国移动、中国联通、中国电信各一台设备或网络。
- Wi-Fi、4G/5G、弱网环境。
- 首次打开、刷新打开、第二天再次打开。
- 横屏和竖屏。
- 点击“开启音乐”后开始前音乐可播放。
- 点击“开始游戏”后游戏中音乐可播放。
- 断网后再次打开，游戏壳能从缓存显示。

目标体验：

- 3 秒内看到游戏画面。
- 未点击音乐前不下载完整音频。
- 控制按钮不遮挡主操作。
- 控制台没有 404、音频路径错误或 service worker 注册错误。
